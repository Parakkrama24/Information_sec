Project Details

Overview

Secure Vehicle-to-Infrastructure (V2I) Communication protocol implementing end-to-end encryption and authentication for vehicle telemetry transmission to edge servers.

Security Features

🔐

Authentication

RSA-based challenge-response mechanism to verify vehicle identity

🔑

Key Exchange

Secure AES session key establishment using RSA-2048 encryption

🛡️

Confidentiality

AES-256-GCM encryption for telemetry data protection

✅

Integrity

Digital signatures ensure data hasn't been tampered with

Protocol Phases

1

Connection Establishment

Vehicle connects to Edge Server via TCP socket

2

Authentication Challenge

Server sends 32-byte cryptographic nonce to vehicle

3

Challenge Response

Vehicle signs challenge with private RSA key and returns signature

4

Session Key Exchange

Vehicle generates AES-256 key and encrypts with server's public RSA key

5

Secure Transmission

Vehicle encrypts telemetry with AES-GCM, signs payload, and transmits

Protocol Security Guarantees

🔐

Mutual Authentication

Vehicle proves identity through RSA digital signature of server's challenge, ensuring only authorized vehicles can connect.

🔑

Secure Key Exchange

AES session key is securely transmitted using RSA-2048 encryption, preventing eavesdropping and man-in-the-middle attacks.

🛡️

Data Confidentiality

All telemetry data is encrypted with AES-256-GCM, ensuring that only the intended recipient can read the information.

✅

Message Integrity

RSA digital signatures and AES-GCM authentication tags ensure data hasn't been tampered with during transmission.

File Descriptions

🔑

keys_setup.py

Generates RSA-2048 key pairs for both vehicle and edge server. Creates PEM-encoded public and private keys used for authentication and encryption.

🖥️

edge_server.py

TCP server implementation that handles vehicle connections, authentication, key exchange, and secure telemetry reception. Acts as the infrastructure endpoint.

🚗

vehicle.py

Client application simulating a vehicle. Performs authentication, generates session keys, and transmits encrypted telemetry data to the edge server.

🧪

test_system.py

Automated test script that starts the edge server and vehicle client, executes the full protocol, and displays results from both components.

📦

requirements.txt

Python dependencies list. Contains the cryptography library for RSA and AES operations.

📝

*.pem files

RSA key files generated by keys_setup.py. Includes private and public keys for both vehicle and edge server.
